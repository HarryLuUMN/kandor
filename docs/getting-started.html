Notebook examples

Introduction: Kandor includes starter notebooks for world creation, temporal KG retrieval, and widget-based inspection.

Usage:

  • examples/notebooks/01_quickstart_world_pipeline.ipynb for the end-to-end mock pipeline
  • examples/notebooks/02_temporal_kg_queries.ipynb for memory and retrieval APIs
  • examples/notebooks/03_temporal_kg_widget.ipynb for notebook visualization of a saved snapshot
  • examples/notebooks/04_openai_world_pipeline.ipynb for the end-to-end pipeline with a real OpenAI-compatible backend

Current status: these notebooks are starter examples for local exploration; the widget notebook requires the optional widget extras; the OpenAI notebook expects OPENAI_API_KEY and optional OPENAI_MODEL and OPENAI_BASE_URL from the environment or the repository .env.

Run the OpenAI smoke example

PYTHONPATH=. python3 examples/smoke_openai.py

Environment variables: OPENAI_API_KEY, optional OPENAI_MODEL, optional OPENAI_BASE_URL.

Minimal usage

from kandor import LoreGenerator, SimulationRunner, WorldBuilder
from kandor.llm.mock import MockLLM

llm = MockLLM(responses={
    "god_agent": {
        "premise": "A broken empire of sandglass cities.",
        "genre": "fantasy",
    }
})

builder = WorldBuilder(llm=llm)
blueprint = builder.create_world(
    prompt="Make a desert empire with unstable time magic."
)
memory = builder.create_memory(blueprint)
runner = SimulationRunner(world=blueprint.spec, memory=memory)
lore = LoreGenerator(llm=llm)
