Core flow

  1. GodAgent compiles a user prompt into a WorldBlueprint.
  2. WorldBuilder turns that blueprint into temporal KG memory.
  3. SimulationRunner advances the world by selecting and applying events.
  4. TemporalKGMemory stores events and relations over time.
  5. LoreGenerator turns structured state into readable summaries.
  6. TemporalKGWidget can inspect exported snapshots visually inside notebooks.

Design principle

The LLM proposes or summarizes; the library owns validation, state updates, memory writes, and retrieval. That separation keeps the system inspectable and testable.

Provider boundary

The llm layer keeps model access swappable. Use MockLLM for local, deterministic tests and OpenAICompatibleLLM when you want the same structured interface backed by a real chat-completions endpoint.
